# product.md — D0 Trending Radar 的判断宪法

## 产品定位

我们在做 **D0（Day Zero）**，核心用户是**加密货币交易者 + AI 重度用户**（Gen-Z crypto 原生用户，习惯用 Telegram Bot / CLI / AI 对话操作链上资产）。

解决的核心问题是：**链上数据工具太分散，AI 无法直接执行链上操作**。D0 将 99 个工具统一成一个 CLI + AI 交互入口，并提供云端托管的专属 Telegram Bot。

目前处于**冷启动 / 早期增长阶段**，核心关注 **D0 激活**（注册 → Onboarding → 执行第一条命令）和**持续使用留存**。

---

## 趋势筛选标准

### 优先收录（高权重）

- "一句话/一条命令搞定复杂链上操作"的内容 hook
- Gen-Z 从 App 迁移到 Bot / CLI / AI 对话的工具选择新模式
- Telegram Bot 作为生活方式工具的案例（真正执行操作，不只是通知）
- AI × 链上工具结合的产品叙事（AI 下单、AI 监控持仓）
- Crypto 社区 FOMO 传播机制（哪些格式让用户转发分享）
- 可借鉴到 Onboarding / 激活路径 / 第一次命令执行 的叙事格式
- 揭示 Gen-Z 在信息获取/工具选择/社交动机 上的新模式

### 降权（低信号）

- 纯娱乐 meme、NFT PFP 文化（与 D0 用户场景无关）
- 品牌营销/KOL 软文（广告感强）
- 超过 2 周的旧趋势（已过传播峰值）
- 纯价格喊单、市场情绪内容（与产品无关）

---

## 输出要求

每条趋势必须包含：

1. **趋势描述** — 这是什么，正在发生什么（1-2句）
2. **用户心理** — Gen-Z crypto 用户为什么消费/传播这个
3. **产品启发** — 必须落地到 D0 的具体产品问题：
   - Onboarding / 激活路径
   - Bot 交互设计
   - 入金流程
   - CLI 命令设计
   - 内容/增长策略
   格式示例：
   - "这是否意味着我们的 onboarding 应该……"
   - "这个 format 能否用在我们的……"
4. **代表链接** — 一个最能说明问题的推文 URL
